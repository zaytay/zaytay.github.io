Solr 与 Sphinx 对比

Lucene Solr Sphinx 三者概念上分析

首先是Solr 与 Lucene 的关系:
Lucene是一个基于Java的全文搜索引擎。 一套信息检索的工具包。 Solr 是一个基于lucene的搜索引擎系统，可以理解为一个搜索服务器。基于lucene的搜索引擎系统还有ElasticSearch、IndexTan、Katta等等。

Sphinx 相对于Lucene来进行定义的比较，它是一个C++的全文检索引擎。

Lucene 与Solr 对比

首先Solr是基于Lucene做的，Lucene是一套信息检索工具包，但并不包含搜索引擎系统，它包含了索引结构、读写索引工具、相关性工具、排序等功能，因此在使用Lucene时仍需要关注搜索引擎系统，例如数据获取、解析、分词等方面的东西。它只是一个工具包，不支持分布式，所以分布式的搜索服务，都需要自己去实现。
Solr的目标是打造一款企业级的搜索引擎系统，因此它更接近于我们认识到的搜索引擎系统，它是一个搜索引擎服务，通过各种API可以让你的应用使用搜索服务，而不需要将搜索逻辑耦合在应用中。而且Solr可以根据配置文件定义数据解析的方式，更像是一个搜索框架，它也支持主从、热换库等操作。还添加了飘红、facet等搜索引擎常见功能的支持。
因此Lucene 需要投入的精力大， Solr是在Lucene外层加上了一层优化。
Lucene在使用上更加灵活，但是你需要自己处理搜素引擎系统架构，以及其他附加功能的实现。而Solr帮你做了更多，但是是一个处于高层的框架，Lucene很多新特性不能及时向上透传，所以有时候可能发现需要一个功能，Lucene是支持的，但是Solr上并没有相关的接口。
从功能上讲，Solr会自带很多功能，而lucene需要自己动手去实现。但对于深度定制的功能需求，lucene修改的灵活性更大。
从性能上讲，因为Solr是基于Lucene的两者差别不大。因为Lucene需要多方面的自定义实现，所以性能的差别一般是由Lucene自定义开发的细节来决定

Solr 与 Sphinx 对比

应用环境：
首先Solr是由Java编写的，Sphinx是由C++编写的。所以对于Java 应用，Solr可以很容易嵌入到应用中。Sphinx也提供有多种语言的接口。
Sphinx的设计初衷，就是与SQL数据库可以有效结合。相对于Solr，Sphinx最大的特点有两个： 1，支持直接以SQL数据库作为数据源
2，支持以SQL语法进行查询 Sphinx 跟 RDBMS （特别是MYSQL） 绑定的特别紧密。 而Solr 可以和 Hadoop
集成，成为分布式系统。 也可以 和 Nutch集成，成为一个功能完备的搜索引擎，以及网络爬虫(crawler) 。

索引： Solr是基于Lucene的索引结构，所以Solr和Sphinx的索引结构并不兼容。
Solr支持近实时更新索引，Sphinx不支持，Sphinx只能通过定时任务进行索引更新。 在Sphinx索引中，所有的 Doment
id 必须是 unique , unsigned, non-zero
整数。Solr的索引结构与lucene兼容，很多索引操作甚至不需要unique key。 而且unique key
可以是整数，也可以是字符串。 因为Sphinx直接跟数据库对话，所以建立索引速度非常快，除非SQL语句非常复杂或者某个列没有索引。
搜索相关功能点： Solr 可以索引像WORD， PDF这样的形式，Sphinx并不支持。 Solr 自带有拼写检查器，Sphinx不支持。
Solr 默认有facet的支持，Shphinx中需要做一些额外的工作才可以支持。 Solr 支持field collapsing
来避免相似搜索结果的重复性。 Sphinx不支持该功能。 Solr支持多核（multicore）形式，Sphinx未发现该功能。 Solr
的SolrCloud 提供 分片(sharding) 、备份（replication）这些功能。Sphinx 并未发现相关功能。

性能效率： 参考 http://blog.csdn.net/wgw335363240/article/details/37931941
的分析
1.索引效率
Sphinx：10-15MB/秒， 实测最高可对100GB的文本建立索引，单一索引可包含1亿条记录
Solr:10MB/秒，亿条数据,20G索引，新浪目前也采用lucene检索
2.搜索性能
Sphinx：高性能搜索，在2-4GB的文本数据上，平均每次检索响应时间小于0.1秒;在1.2G文本，100万条文档上进行搜索，支持高达每秒150~250次查询；
Solr:高性能搜索，8G的索引文件，检索响应时间为150ms。高峰支持500并发/秒

索引、搜索的具体性能有待进一步实验平台测试。
